Introduction

The language described in this book is The ECMAScript Programming Language, Fifth Edition. The standard, which is designated as ECMA-262, may be found at http://www.ecma-international.org/publications/standards/ECMA-262.HTM. The Fifth Edition is sometimes referred to as ES5. Because of trademark problems, the standard for the JavaScript language is called ECMAScript. JavaScript is a trademark of Oracle Corporation.

The Third Edition is sometimes referred to as ES3. Paragraphs marked with the ES3 icon in the left margin indicate concerns for users of The Third Edition. The Fifth Edition significantly improved the overall quality of the language, but that may be of little comfort to the users of The Fifth Edition. The Fifth Edition is implemented in all popular web browsers at the time of publication of this book, and the new features or corrections specified in The Fifth Edition may not yet be available. Programmers who must honor compatibility commitments should pay close attention to the ES3 warnings. (There is no Fourth Edition.)

The Fifth Edition (ES5) describes two dialects, the new strict dialect and the traditional default dialect. You should either be using the ES5 strict dialect for reliability, or the ES3 dialect for compatibility. There is no reason to use the ES5 default dialect, so the ES5 default dialect is not included in this book. If you are using the ES5 strict dialect, add this line to the top of every script file:

"use strict";

The use strict pragma is written as a string literal so that it will not cause a syntax error when processed by an ES3 system.

Most of the material in this encyclopedia is organized into 27 divisions (one for each letter and the special characters). Each division lists several specimens, where a specimen is an element of syntax or a concept. The specimens are ordered alphabetically. A specimen marked with reserved word indicates that the word is reserved and cannot be used as a parameter or variable name. The special characters division is sorted according to the ASCII sequence.

Under each specimen there will be one or more articles that explain, expand, or are related to the specimen. An article may be composed of one or more parts.

The title of an article may contain one or more classifiers, such as

  • function
  • global
  • infix operator
  • literal
  • prefix operator
  • prototype
  • regexp operator
  • separator
  • statement
  • suffix operator
  • ternary operator

There is an article for each classifier.

The article of a title may contain a comment. It is used in cases where the subject is an abbrievation or special character.

Many articles will have examples. Expression examples will be written with a comment showing the result. 

3 + 4      // 7
3 + '4'    // "34"

Chapter 16 of the ECMAScript standard gives implementors of ECMAScript processors some license to extend the language with experimental or proprietary features. Such features might be incorporated into a future edition or might be abandoned. It is unwise to rely on such features because of the likelihood of diminished reliability. Such non-standard features will not be included in this encyclopedia.
